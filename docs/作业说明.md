## 作业说明

**实现目标**：
基于 React 或 Vue 框架，结合 Ant Design 或 Element UI 组件库，开发一款功能完整、体验流畅的电商平台商品列表页。核心目标是实现商品的高效筛选、排序与分页展示，通过 Redux Toolkit 或 Pinia 进行统一状态管理，同时兼顾性能优化与用户体验提升，最终交付一个符合电商场景实际需求、可复用性强的前端页面方案。

**功能列表**：

基础功能：

1. 技术栈选型：选用 React/Vue 框架 + Ant Design/Element UI 组件库，搭配 Redux Toolkit/Pinia 实现全局状态管理；
2. 商品列表展示：模拟数据交互（使用 Mock.js 生成商品数据），以卡片或网格布局呈现商品核心信息（含商品图片、名称、价格、销量等）；
3. 多条件筛选：支持按分类、价格区间、销量、评分等维度进行组合筛选，筛选状态实时同步；
4. 灵活排序：提供价格（升序 / 降序）、销量（降序）、上架时间（最新 / 最热）等排序选项，切换后列表实时更新；
5. 分页功能：实现分页控制（含页码切换、每页条数选择），支持快速跳转至指定页码，分页状态与列表数据联动。

进阶功能：

1. 筛选防抖处理：对价格输入、关键词搜索等筛选操作添加防抖逻辑（建议 300-500ms），避免短时间内频繁触发数据请求，减轻接口压力；
2. 加载状态优化：列表数据请求过程中显示骨架屏（Skeleton），替代传统加载动画，提升用户等待体验；
3. 大数据量性能优化：集成虚拟滚动技术（如 react-virtualized、vue-virtual-scroller），当商品数据量超过一定数量时自动启用，减少 DOM 节点渲染，提升滚动流畅度；
4. AI 推荐模块：在列表页顶部添加 “猜你喜欢” 区域，模拟个性化推荐逻辑（可基于本地模拟数据、用户浏览痕迹或固定推荐列表实现），展示 4-8 个推荐商品卡片。

自由探索：
例如：

1. 筛选条件记忆：通过 localStorage 或状态管理保存用户筛选 / 排序偏好，页面刷新后自动恢复上次状态；
2. 商品卡片交互：添加 hover 效果（如阴影加深、轻微放大）、快速加入购物车 / 收藏功能；
3. 响应式适配：开发响应式 UI，页面在电脑、平板、手机端均有良好展示效果；
4. 空状态与异常处理：当筛选结果为空时显示友好提示，支持重置筛选条件；请求失败时提供重试按钮；
5. 主题切换：支持明暗主题切换，商品卡片、筛选组件等样式同步适配；
6. 搜索联想：在关键词搜索框中添加搜索联想 / 历史记录功能，提升搜索效率。

**评估标准**：

- 功能完整性：基础功能（筛选、排序、分页、状态管理）全部实现，逻辑无漏洞；
- 技术规范性：
  - 框架与组件库使用规范，代码结构清晰（如组件拆分合理、状态管理逻辑简洁）；
  - 防抖、虚拟滚动等技术实现正确，无性能问题；
  - 代码符合 ES6+ 规范，命名统一，注释清晰；
- 用户体验：
  - 页面布局美观，响应式适配良好；
  - 骨架屏、加载状态、空状态等反馈及时，交互流畅；
  - 筛选 / 排序切换无卡顿，分页操作便捷；
- 文档表达：README + 视频讲解结构清晰
- 创新与拓展：
  - 自由探索功能每实现 1 项；
  - 功能设计贴合电商实际场景，具有实用性和复用性，可额外加分。
