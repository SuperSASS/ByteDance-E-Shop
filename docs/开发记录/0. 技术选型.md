## 有关 UI 的选型

个人梳理总结了下，目前 UI 主要分为三类：

- CSS 层级
  这些都是最基础的 CSS 技术，最后都会转化为 CSS，只在这一层级要自己从 0 手写样式
  - 原生 CSS：CSS3
  - CSS 预处理：Sass/Less，原生 CSS 的语法扩展，最后编译为 CSS
  - CSS-in-JS：Emotion/Styled-components，在 JS 中写 CSS（也是多数组件库用于自定义样式的方式（如 MUI 用 Emotion，AntD 用自研的 CSS-in-JS））
  - Utility-first CSS：Tailwind CSS，通过`class`、组合 Utility（原子）类来构建样式，相当于对原生 CSS 的封装
- Headless UI 层级  
  在这一层级以组件的形式组织，但**只关注逻辑、不关注样式**（即样式还是要自己从 0 手写），  
  比如封装了`<Card>`、`<Dialog>`这些组件，提供逻辑（点按钮会弹出 Dialog 框），但没有样式（不写就是浏览器默认样式）  
  最经典就是 Radix Primitives（注意不是 Radix Themes）
- UI 层级  
  这一层级就是含“逻辑 + 样式”的组件，即我们日常用的 UI 组件库，有两种类别
  - 成熟 UI 组件库：AntD、Element UI、MUI、Joy UI、Radix Themes 等
    - 有统一样式的组件，但组件作为黑盒、npm 包，对着接口使用
    - 开发小型项目时，你甚至不用写一个`components`。
    - 自定义能力差（因为没有源码），只能基于文档提供的方式自定义，或者用强制魔改的方式
  - 预制 UI 组件库：Shadcn UI（Radix Primitives（逻辑） + Tailwind CSS（样式））
    - 与上者最大的区别是：**提供源码**（相当于你自己写了这个组件，复制粘贴的源码到本地项目）
    - 也带基本的样式，你也可以不用写一个`components`。
    - 因为有源码，可任意修改自定义。
